# 6.3 Telas Principais

## Introdução

Este documento apresenta as telas mais significativas do sistema Digi Eventos, descrevendo seus objetivos e a dinâmica de navegação. O objetivo é demonstrar o produto final e a experiência do usuário.

---

## 1. Tela de Login

![Placeholder para Tela de Login](https://via.placeholder.com/800x450.png/f0f0f0/333333?text=Tela+de+Login)

**Objetivo**: Permitir que usuários existentes acessem o sistema.

**Dinâmica de Navegação**:
- O usuário insere seu e-mail e senha.
- Campos possuem validação em tempo real (formato de e-mail, senha não vazia).
- Ao clicar em "Entrar", o sistema valida as credenciais no backend.
- **Sucesso**: O usuário é redirecionado para o Dashboard (`/app/dashboard`).
- **Falha**: Uma mensagem de erro "Credenciais inválidas" é exibida.
- Há um link para a página de criação de conta para novos usuários.

---

## 2. Tela Inicial (Listagem de Eventos)

![Placeholder para Listagem de Eventos](https://via.placeholder.com/800x450.png/f0f0f0/333333?text=Listagem+de+Eventos)

**Objetivo**: Apresentar todos os eventos disponíveis para o público geral.

**Dinâmica de Navegação**:
- Exibe uma lista de eventos em formato de cards.
- Cada card mostra informações essenciais: imagem, título, data, local e categoria.
- A tela é pública e não requer autenticação.
- **Navegação**: Clicar em um card de evento leva o usuário para a tela de "Detalhes do Evento".
- O cabeçalho principal permite acesso ao login, registro ou ao dashboard (se já estiver logado).

---

## 3. Tela de Detalhes do Evento

![Placeholder para Detalhes do Evento](https://via.placeholder.com/800x450.png/f0f0f0/333333?text=Detalhes+do+Evento)

**Objetivo**: Fornecer informações completas sobre um evento específico e permitir a inscrição.

**Dinâmica de Navegação**:
- Exibe uma imagem de capa grande, título, descrição detalhada, data, horário, local e capacidade máxima.
- Um botão "Inscrever-se" é exibido para usuários autenticados.
  - Se o usuário não estiver logado, o botão o redireciona para a tela de login.
  - Se o evento estiver lotado ou já tiver ocorrido, o botão é desabilitado.
- Após a inscrição, o botão muda para "Cancelar Inscrição".
- Para administradores, botões de "Editar Evento" e "Excluir Evento" são exibidos.

---

## 4. Dashboard (Área do Usuário)

![Placeholder para Dashboard](https://via.placeholder.com/800x450.png/f0f0f0/333333?text=Dashboard+do+Usuário)

**Objetivo**: Servir como a página central para usuários autenticados, oferecendo acesso rápido às principais funcionalidades.

**Dinâmica de Navegação**:
- Acessível apenas para usuários logados.
- Exibe um resumo das atividades do usuário, como "Meus Próximos Eventos".
- Apresenta um menu de navegação lateral com links para:
  - **Dashboard**: Página inicial da área logada.
  - **Meu Perfil**: Para editar informações pessoais.
  - **Gerenciar Eventos** (Admin): Acesso à criação e edição de eventos.
  - **Gerenciar Usuários** (Admin): Acesso à lista de usuários.
  - **Gerenciar Categorias** (Admin): Acesso ao CRUD de categorias.

---

## 5. Tela de Perfil do Usuário

![Placeholder para Perfil do Usuário](https://via.placeholder.com/800x450.png/f0f0f0/333333?text=Perfil+do+Usuário)

**Objetivo**: Permitir que o usuário visualize e atualize suas informações pessoais e de segurança.

**Dinâmica de Navegação**:
- Exibe os dados atuais do usuário: nome, e-mail e avatar.
- Permite a atualização do nome e do avatar (com upload de imagem).
- Inclui um botão "Alterar Senha", que abre um diálogo para que o usuário insira a senha atual e a nova senha.
- O e-mail não é editável.

---

## 6. Telas de Administração

As telas de administração são acessíveis apenas para usuários com o perfil "admin" e permitem o gerenciamento completo do sistema.

### 6.1 Gerenciamento de Eventos

![Placeholder para Gerenciamento de Eventos](https://via.placeholder.com/800x450.png/f0f0f0/333333?text=Gerenciamento+de+Eventos+(Admin))

**Objetivo**: Permitir que administradores criem, editem e excluam eventos.

**Dinâmica de Navegação**:
- Exibe uma tabela com todos os eventos (ativos e cancelados).
- A tabela inclui opções de busca e filtro.
- Cada linha da tabela possui botões para "Editar" e "Excluir".
- Um botão "Criar Novo Evento" leva a um formulário completo para adicionar um novo evento, incluindo upload de imagem.

### 6.2 Gerenciamento de Usuários

![Placeholder para Gerenciamento de Usuários](https://via.placeholder.com/800x450.png/f0f0f0/333333?text=Gerenciamento+de+Usuários+(Admin))

**Objetivo**: Permitir que administradores gerenciem todos os usuários do sistema.

**Dinâmica de Navegação**:
- Apresenta uma tabela com a lista de todos os usuários, seus e-mails e perfis (role).
- Permite ao administrador alterar o perfil de um usuário (de "user" para "admin" e vice-versa).
- Oferece a funcionalidade de redefinir a senha de um usuário.
- Um botão para "Excluir Usuário" está disponível.
